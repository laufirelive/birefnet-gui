# BiRefNet GUI 视频抠图工具 - 设计文档

## 1. 项目概述

### 1.1 项目名称
**BiRefNet Video Matting Tool** (暂定)

### 1.2 目标
创建一个 Windows 平台开箱即用的视频抠图工具，基于 BiRefNet 模型，提供简洁的图形界面。

### 1.3 核心特性
- 一键安装，无需配置 Python 环境
- 支持视频文件输入（MP4、AVI、MOV 等）
- 支持图片序列输入
- 实时预览抠图效果
- 多种输出格式（透明视频、绿幕、蒙版等）
- 硬件加速支持（CUDA/TensorRT）

## 3. 功能设计

### 3.1 核心功能模块

#### 3.1.1 输入模块
| 功能 | 说明 |
|------|------|
| 视频文件输入 | 支持 MP4、AVI、MOV、MKV 等常见格式 |
| 图片序列输入 | 支持文件夹选择，自动识别序列 |

#### 3.1.2 处理模块
| 功能 | 说明 |
|------|------|
| 模型选择 | 6种模型自由切换，按需下载 |
| 显存检测 | 自动检测显卡，提示模型兼容性 |
| 分辨率自适应 | 根据显存自动选择处理分辨率 |
| 批处理 | 多帧并行推理，提升吞吐量 |
| 时序一致性 | 可选的帧间平滑处理，减少闪烁 |
| ROI 区域选择 | 只处理感兴趣区域，加速处理 |
| 处理范围选择 | 处理全部/指定时间段/标记帧 |
| 视频预览 | 显示原图（不实时推理，减轻性能压力） |
| 音频保留 | 可选保留原视频音轨 |
| 批量队列 | 多任务排队处理，支持拖拽添加 |
| 工程文件 | 保存进度，支持断点续传 |

#### 3.1.3 输出模块

**输出格式：**

| 格式 | 编码 | 透明通道 | 适用场景 |
|------|------|----------|----------|
| MOV | ProRes 4444 | ✅ | 专业后期，质量最高 |
| MOV | PNG 序列 | ✅ | 无损，文件大 |
| WebM | VP9 | ✅ | 网页播放，压缩率高 |
| MP4 | H.264 | ❌ | 兼容性最好 |
| MP4 | H.265/HEVC | ❌ | 相同质量体积更小 |
| MP4 | AV1 | ❌ | 新一代编码，压缩率高 |
| 图片序列 | PNG (RGBA) | ✅ | 逐帧输出，灵活处理 |
| 图片序列 | TIFF | ✅ | 无损，专业后期 |

**输出模式：**

| 模式 | 说明 |
|------|------|
| 透明背景 | 保留 Alpha 通道，直接合成 |
| 绿幕 (Green) | #00FF00 绿色背景 |
| 蓝幕 (Blue) | #0000FF 蓝色背景 |
| 黑底白蒙版 | 输出黑白 Alpha 蒙版 |
| 白底黑蒙版 | 反转蒙版 |
| 原图+蒙版分轨 | 视频左半原图，右半蒙版 |

**高级参数：**

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| 码率 (Bitrate) | 1-500 Mbps | 自动 | 越高质量越好，文件越大 |
| CRF 质量 | 0-51 | 18 | H.264/H.265 质量模式，越低越好 |
| 预设 (Preset) | ultrafast~veryslow | medium | 编码速度与压缩率平衡 |
| 色彩空间 | BT.709/BT.2020 | BT.709 | 广色域内容选 BT.2020 |
| 帧率 | 保持/自定义 | 保持 | 可强制指定输出帧率 |
| 分辨率 | 保持/1/2/1/4/自定义 | 保持 | 可下采样输出 |
| 批处理大小 | 1/2/4/8/自动 | 自动 | 根据显存自动推荐，可手动调整 |

#### 3.1.6 工程文件与断点续传

**工程文件 (.brm)：**
| 保存内容 | 说明 |
|----------|------|
| 队列任务列表 | 所有等待和进行中的任务 |
| 当前任务索引 | 正在执行第几个任务 |
| 输入文件路径 | 原始视频/图片位置 |
| 处理进度 | 当前任务已完成帧数/总帧数 |
| 参数配置 | 每个任务的模型、格式、码率等设置 |
| 临时文件 | 中间结果缓存路径 |
| 输出设置 | 每个任务的目标路径和文件名 |

**缓存目录规范：**
```
Windows: %LOCALAPPDATA%\BiRefNet-GUI\cache\
        例: C:\Users\<用户名>\AppData\Local\BiRefNet-GUI\cache\

结构:
cache/
├── <工程ID_毫秒时间戳>/
│   ├── frames/          # 提取的原始帧
│   ├── masks/           # 生成的蒙版帧
│   └── preview/         # 预览缩略图
└── temp/                # 临时合成文件
```

**清理策略：**
- 工程完成后自动清理对应缓存
- 软件启动时检查并清理超过7天的残留缓存
- 用户可手动清理：设置 → 缓存管理 → 一键清理 |

**工作流程（单任务）：**
1. **开始处理** → 自动创建工程文件
2. **处理中** → 每完成100帧自动保存进度
3. **关闭软件** → 自动保存队列状态
4. **重新打开** → 检测未完成的队列，提供续传选项
5. **完成处理** → 清理临时文件，自动开始下一个任务

**队列断点续传：**
- 整个队列状态自动保存
- 当前任务进度精确到帧（如 100/500 帧）
- 后续任务保持等待状态
- 重启软件后恢复完整队列

**界面体现：**
```
┌─────────────────────────────────────────────────────────────┐
```

### 4.4 模型下载脚本

开发时使用以下脚本准备离线模型包：

```python
#!/usr/bin/env python3
"""
download_models.py - 下载 BiRefNet 模型用于离线打包

运行方式:
    python download_models.py

下载的模型将保存在 ./models/ 目录，可直接打包到安装程序
"""

import os
from huggingface_hub import snapshot_download

# 模型配置列表
MODELS = {
    "birefnet-general": "zhengpeng7/BiRefNet",
    "birefnet-lite": "zhengpeng7/BiRefNet_lite",
    "birefnet-matting": "zhengpeng7/BiRefNet-matting",
    "birefnet-hr": "zhengpeng7/BiRefNet_HR",
    "birefnet-hr-matting": "zhengpeng7/BiRefNet_HR-matting",
    "birefnet-dynamic": "zhengpeng7/BiRefNet_dynamic",
}

def download_model(name, repo_id, save_dir="./models"):
    """下载单个模型"""
    local_path = os.path.join(save_dir, name)
    
    print(f"\n{'='*60}")
    print(f"下载模型: {name}")
    print(f"来源: {repo_id}")
    print(f"保存路径: {local_path}")
    print(f"{'='*60}")
    
    try:
        snapshot_download(
            repo_id=repo_id,
            local_dir=local_path,
            local_dir_use_symlinks=False,  # 复制文件，不使用符号链接
            resume_download=True,           # 支持断点续传
        )
        print(f"✅ {name} 下载完成")
        
        # 显示目录大小
        total_size = get_dir_size(local_path)
        print(f"   大小: {total_size / 1024 / 1024:.1f} MB")
        
    except Exception as e:
        print(f"❌ {name} 下载失败: {e}")

def get_dir_size(path):
    """计算目录大小"""
    total = 0
    for dirpath, dirnames, filenames in os.walk(path):
        for f in filenames:
            fp = os.path.join(dirpath, f)
            total += os.path.getsize(fp)
    return total

def main():
    # 创建模型目录
    os.makedirs("./models", exist_ok=True)
    
    print("BiRefNet 模型下载工具")
    print("=" * 60)
    
    # 下载所有模型
    for name, repo_id in MODELS.items():
        download_model(name, repo_id)
    
    # 汇总信息
    print(f"\n{'='*60}")
    print("下载完成！")
    print(f"{'='*60}")
    
    total_size = get_dir_size("./models")
    print(f"总大小: {total_size / 1024 / 1024:.1f} MB ({total_size / 1024 / 1024 / 1024:.2f} GB)")
    print(f"\n模型保存在: {os.path.abspath('./models')}")
    print("\n打包时将此目录包含进安装程序即可实现离线运行")

if __name__ == "__main__":
    main()
```

## 5. 部署方案

### 5.1 打包策略

```
BiRefNet-Video-Matting/
├── BiRefNet-GUI.exe          # 主程序
├── models/                    # ✅ 预置模型（离线运行）
│   ├── birefnet-general/     # 通用模型 (~400MB)
│   ├── birefnet-lite/        # 轻量模型 (~200MB)
│   ├── birefnet-matting/     # 抠图模型 (~400MB)
│   ├── birefnet-hr/          # 高分辨率模型 (~800MB)
│   ├── birefnet-hr-matting/  # 高分辨率抠图 (~800MB)
│   └── birefnet-dynamic/     # 动态分辨率 (~500MB)
├── ffmpeg/                    # 内置 FFmpeg
├── config.json               # 用户配置

```

**安装包大小预估：**
- 基础程序：~50MB
- 单个模型：~400MB

```python
class VideoMattingPipeline:

        # 2. 帧提取
        frame_generator = ffmpeg.extract_frames(input_path)
        
        # 3. 批量推理
        for batch in batch_iterator(frame_generator, batch_size):
```

### 推理后端说明

**使用 PyTorch 的原因：**

| 优势 | 说明 |
|------|------|
| 开发简单 | BiRefNet 原生支持，无需模型转换 |
| 调试方便 | 可逐层查看中间结果 |
| 生态完整 | 与 transformers 库无缝集成 |
| 维护成本低 | 一套代码，一种推理方式 |

**代价：**
- 安装包体积大（PyTorch ~2GB）
- 启动稍慢（需加载 PyTorch 运行时）

**权衡**：开发效率优先，接受打包体积增大的代价。
